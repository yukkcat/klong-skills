# Security Policy

## API keys

Never include an API key in an issue, pull request, log, screenshot, prompt, or committed file. Revoke and rotate a key immediately if it is exposed.

## Reporting a vulnerability

Do not open a public issue for a vulnerability that could expose credentials or user data. Report it privately through GitHub Security Advisories for this repository.

Include the affected version, reproduction steps, and impact. Do not include real user credentials or data.
